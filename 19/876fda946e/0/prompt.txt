# Plan: Opt-in Model Availability Checking + Stop Button

**Issue**: [#577](https://github.com/moltis-org/moltis/issues/577) — Model availability probing runs automatically after provider setup, consuming tokens/quota without user consent. No way to stop it.

**Goal**: Make detection opt-in only (user clicks a button) and add a "Stop" button during active detection.

---

## Part A: Remove automatic background probing

### Backend — `crates/gateway/src/methods/services.rs`

Remove 4 `tokio::spawn` blocks that auto-trigger `detect_supported`:

| Location | Trigger | Lines |
|---|---|---|
| After `providers.save_key` | Credentials saved | 3358-3365 |
| After `providers.oauth.start` (already auth'd) | OAuth shortcircuit | 3401-3413 |
| After `providers.oauth.complete` | OAuth flow done | 3450-3456 |
| After `providers.save_models` | Preferred models saved | 3493-3499 |

Also remove the `model_probe_params()` helper (lines 17-28) — no remaining callers.

Clean up now-unused `provider_name` bindings in `save_key`, `oauth.start`, and `oauth.complete` handlers (they were only used for probe params). Keep the one in `save_models` if used by the `save_models` call itself (check — it's extracted from params but only used for probing, so remove it too).

### Frontend

| File | Lines | What to remove |
|---|---|---|
| `crates/web/src/assets/js/providers.js` | 1124-1128 | `sendRpc("models.detect_supported", ...)` after OAuth when no models found |
| `crates/web/src/assets/js/onboarding-view.js` | 1563-1567 | Same pattern in onboarding flow |

After removal, these paths just show the success state and let the user discover models manually via "Detect All Models" on the LLMs page.

## Part B: Add cancellation support

### `crates/chat/Cargo.toml`
Add `tokio-util = { workspace = true }` (already in workspace deps as `"0.7"`).

### `crates/chat/src/lib.rs`

1. **New field** on `LiveModelService`:
   ```rust
   detect_cancel: Arc<tokio::sync::RwLock<Option<CancellationToken>>>,
   ```
   Initialize as `None` in `new()`.

2. **In `detect_supported()`** (after acquiring `_run_permit`):
   - Create a new `CancellationToken`, store it in `detect_cancel`
   - Change the main probe loop from `while let Some(joined) = tasks.next().await` to a `tokio::select!` that also watches for cancellation
   - On cancel: break, broadcast `"cancelled"` phase, clear token, return partial results with `"cancelled": true`
   - On normal completion: clear token (existing "complete" broadcast stays)

3. **New method `cancel_detect()`**:
   - Read the stored token, call `.cancel()` if present
   - Return `{ ok: true, cancelled: true/false }`

### `crates/service-traits/src/lib.rs`

Add `cancel_detect(&self) -> ServiceResult` to `ModelService` trait + `NoopModelService` impl (returns ok, cancelled: false).

### `crates/graphql/tests/integration.rs`

Add `cancel_detect` to `MockModel` impl (line ~648): delegates to `self.0.call("models.cancel_detect", json!({}))`.

### `crates/gateway/src/methods/services.rs`

Register `"models.cancel_detect"` RPC (after `models.detect_supported` registration).

### `crates/gateway/src/methods/mod.rs`

Add `"models.cancel_detect"` to `WRITE_METHODS` (after line 175, next to `models.test`).
