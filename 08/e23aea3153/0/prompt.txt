# Fix Gemini thought_signature round-tripping (#375) and schema validation (#793)

## Context

**#375**: Gemini 3.x models return `thought_signature` on each tool call in their OpenAI-compat
API responses. When Moltis replays tool calls in subsequent turns, this field must be present or
Gemini rejects with HTTP 400. Currently `ToolCall` only carries `id`, `name`, `arguments` -- the
signature is silently dropped at parse time. This affects both direct Gemini API and OpenRouter routing.

**#793**: When OpenRouter routes to Google AI Studio (not Vertex), tool schemas fail validation
because `required` arrays reference properties that Google AI Studio considers undefined. The
existing `PruneOrphanedRequiredTransform` (added for #747) doesn't catch all edge cases -- likely
properties with type-only or description-only schemas that AI Studio treats as undefined.

These interact: users can't work around #375 via OpenRouter because #793 blocks that path.

## Approach: provider-metadata passthrough on ToolCall

Add an opaque `metadata` field to `ToolCall` that preserves any provider-specific fields (like
`thought_signature`) through the full round-trip: API response -> typed struct -> persistence ->
deserialization -> next API request.

Use an allowlist of known metadata keys to avoid bloating session storage. Start with
`["thought_signature"]`; extensible for future provider-specific fields.

### 2. Extract metadata from Gemini API responses

**`crates/providers/src/openai_compat/provider.rs`**:

Add helper (near `parse_tool_calls`, line 263):
```rust
const TOOL_CALL_METADATA_KEYS: &[&str] = &["thought_signature"];

fn extract_tool_call_metadata(tc: &serde_json::Value) -> Option<serde_json::Map<String, serde_json::Value>> {
    let obj = tc.as_object()?;
    let meta: serde_json::Map<_, _> = TOOL_CALL_METADATA_KEYS.iter()
        .filter_map(|&k| obj.get(k).map(|v| (k.to_string(), v.clone())))
        .collect();
    if meta.is_empty() { None } else { Some(meta) }
}
```

**Non-streaming** (`parse_tool_calls`, line 263): call `extract_tool_call_metadata(tc)` and set on `ToolCall`.

**Streaming** (`process_openai_sse_line`, ~line 670): extract metadata from the first SSE chunk of
each tool call (where `id` and `function.name` appear). Add `tool_call_metadata: HashMap<usize, Option<Map>>>`
to `StreamingToolState`. Emit in `ToolCallStart`.

### 6. Strengthen schema normalization for #793

**`crates/providers/src/openai_compat/schema_normalization.rs`** -- `PruneOrphanedRequiredTransform`:

Tighten the "defined property" check. Currently treats any non-empty, non-boolean schema as defined.
Add a `has_usable_type` check -- a property must have `type`, `enum`, `anyOf`, `oneOf`, `allOf`, or
`$ref` to be considered defined:
```rust
fn has_usable_type(v: &serde_json::Value) -> bool {
    let Some(obj) = v.as_object() else { return false };
    ["type", "enum", "anyOf", "oneOf", "allOf", "$ref"]
        .iter().any(|k| obj.contains_key(*k))
}
```

Also add a post-normalization `EnsurePropertyTypeTransform` that infers `"type": "string"` for
properties in `required` that have a description but no type info. This is safer than pruning the
property because it preserves the parameter name for the LLM.
