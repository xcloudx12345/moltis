# Fix: MiniMax system role rejection (issue #592)

## Context

MiniMax API rejects `role: "system"` in the messages array with error 2013.
This was previously fixed in commit `1bf8e9b9` (closed #508) by extracting
system messages into a top-level `"system"` field in the request body.
PR #586 (`f96dcc46`) incorrectly reverted that fix — it removed
`requires_top_level_system_prompt()` and `prepare_request_messages()`,
putting system messages back as `role: "system"` in the messages array.
The PR was only verified with mocks, not against the real MiniMax API.

## Approach

Restore the system prompt extraction for MiniMax, following the same pattern
used by the Anthropic provider (`to_anthropic_messages()` in `anthropic.rs`)
and the Responses API (`split_responses_instructions_and_input()` in
`openai_compat.rs`): extract all system messages from the messages array,
join their content with `\n\n`, and place the result in a top-level `"system"`
field on the request body.

2. **Re-add `extract_system_prompt()` method** (after `is_minimax`):
   ```rust
   /// For providers that reject `role: "system"` in the messages array
   /// (e.g. MiniMax), extract all system messages and return them as a
   /// joined string alongside the remaining messages.
   fn extract_system_prompt(
       &self,
       messages: Vec<serde_json::Value>,
   ) -> (Vec<serde_json::Value>, Option<String>) {
       if !self.is_minimax() {
           return (messages, None);
       }
       let mut system_parts = Vec::new();
       let mut out = Vec::with_capacity(messages.len());
       for message in messages {
           if message.get("role").and_then(Value::as_str) == Some("system") {
               if let Some(content) = message.get("content").and_then(Value::as_str) {
                   if !content.is_empty() {
                       system_parts.push(content.to_string());
                   }
               }
               continue;
           }
           out.push(message);
       }
       let system_prompt = (!system_parts.is_empty()).then(|| system_parts.join("\n\n"));
       (out, system_prompt)
   }
   ```

## Edge cases

- **No system messages**: `extract_system_prompt` returns `None`, body unchanged
- **Empty system content**: Skipped (not added to `system_parts`)
- **Multiple system messages at different positions**: All extracted and joined
- **Non-MiniMax providers**: `is_minimax()` returns false, pass-through unchanged
- **Custom provider with MiniMax base URL**: Detected via URL contains check
