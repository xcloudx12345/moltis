Reading Discord I got:

Hey, I just upgraded an instance to moltis from openclaw and I'm struggling to setup the telegram bot, I wanted to use this config :
[channels.telegram.bot]
token = "tokentelegram"
allowed_users = [ids_interger, ...]
dm_policy = "allowlist"

But I have to approve the sender for every new session in the webui, is that how it's supposed to work ? Is there a way to pair a user to the bot indefinitely ? I tried with the name of the user, without the dm_policy and a few different things and now I'm a bit lost 


My bot told me this:
The correct config

TOML[channels.telegram.my-bot]
token = "tokentelegram"
dm_policy = "allowlist"
allowlist = ["123456789", "987654321"]   # Telegram user IDs as STRINGS

Key differences from what they tried:
allowlist not allowed_users
Values are strings, not integers (the allowlist is Vec<String>, matching is case-insensitive and supports globs)
Can use either numeric user IDs or usernames (both are checked in check_dm_access)

(Just clone the moltis repo to your local machine, ask moltis to search in the source code and find the answre)
