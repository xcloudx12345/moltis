# Lazy Tool Registry — Implementation Plan

## Context

Every LLM turn includes full JSON schemas of all registered tools. With a few MCP servers connected this easily burns 15,000+ tokens per turn before the model reads a single user message. PR #330 proposed a solution but was too large (72K additions, mostly unrelated). This plan extracts the core lazy-tool idea and implements it cleanly.

## Design

When `registry_mode = "lazy"` is set in config, the model sees one meta-tool (`tool_search`) instead of all tool schemas:

- `tool_search(query="memory")` -> name + description pairs, no schemas, max 15 results
- `tool_search(name="tool_name")` -> full schema returned, tool **activated** in session
- On next iteration, model calls tool directly — standard runner pipeline, hooks fire normally

Key mechanism: `ToolRegistry` gains an `activated: Arc<Mutex<HashMap<String, Arc<dyn AgentTool>>>>` field (using `std::sync::Mutex`, NOT tokio — lock held for microseconds, keeps `list_schemas()` sync). `ToolSearchTool` shares this Arc and inserts entries when activating. Runner re-computes schemas each iteration so activated tools appear immediately.

**ToolRegistry changes:**
1. Add `pub(crate) activated: Arc<Mutex<HashMap<String, Arc<dyn AgentTool>>>>` field
2. Initialize empty in `new()` — always present (simpler than Option)
3. `get_arc()`: check `tools` first, then `activated` map
4. `list_schemas()`: append schemas from `activated` map after static tools
5. Add `list_names()` update to include activated tools
6. All `clone_*` methods: new registry gets fresh empty `activated` (sub-agents don't inherit)

**Runner changes (both `run_agent_loop_with_context` and `run_agent_loop_streaming`):**
1. Move `let tool_schemas = tools.list_schemas()` from before the loop to inside it
2. Move `schemas_for_api` computation inside the loop
3. Change `tools.get()` to `tools.get_arc()` for tool dispatch (checks activated map)
4. When registry has activated tools mechanism, multiply `max_iterations` by 3

### Phase 2: ToolSearchTool + wrap_registry_lazy

**Files:**
- `crates/agents/src/lazy_tools.rs` (NEW)
- `crates/agents/src/lib.rs` — add `pub mod lazy_tools`

**ToolSearchTool:**
```rust
pub struct ToolSearchTool {
    full_registry: Arc<ToolRegistry>,
    activated: Arc<Mutex<HashMap<String, Arc<dyn AgentTool>>>>,
}
```

- Keyword search: match query against all tool names/descriptions, return top 15 as `{name, description}` pairs (no schemas)
- Name activation: look up in `full_registry`, insert into `activated`, return full schema + hint
- Scoring: name match (100) > name contains (50) > word overlap (10)

**`wrap_registry_lazy(full: ToolRegistry) -> ToolRegistry`:**
1. Wrap `full` in `Arc`
2. Create new empty `ToolRegistry`
3. Share the new registry's `activated` Arc with `ToolSearchTool`
4. Register `ToolSearchTool` into the new registry
5. Return the new registry (contains only `tool_search`, but `ToolSearchTool` can search/activate from `full`)
